The README renders these four focused cards in a 2×2 grid. Each card
uses fixed metric tiles and a computed evidence-depth indicator, so
labels and final rows cannot be clipped.

### Context Engineering

Detects instruction hierarchy, architecture decisions, handovers,
context selection and token budgets, context compression, and
retrieval-based context injection.

### Memory Engineering

Detects memory implementation files, checkpoint savers, persistent
state, sessions and conversation memory, long-term memory, semantic or
episodic memory, and recognized memory frameworks.

### AI Harness Engineering

Detects runners and execution loops, tool registries, lifecycle hooks,
retry and recovery policies, timeout or token budgets, model routing
and fallback, concurrency, sandboxing, structured output, tracing, and
CI validation.

### Agentic Orchestration

Detects agent definitions, multi-agent architecture, router/director/
supervisor agents, planning and delegation, handoffs, parallel or
sequential workflows, and human-in-the-loop checkpoints.

## Evidence and privacy boundaries

Generic AI keywords are insufficient by themselves. Cards do not claim
AI-generated-code percentages, prompt volume, productivity gains, or
code-quality gains.

Private repository names, prompt contents, agent instructions,
credentials, MCP URLs, and environment values are never rendered.
